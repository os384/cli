# Changelog

All notable changes to `cli` are documented here.

Format: [Keep a Changelog 1.1.0](https://keepachangelog.com/en/1.1.0/)
Versioning: [Semantic Versioning 2.0.0](https://semver.org/spec/v2.0.0.html)

## [Unreleased]

### Changed

- **Three-file config split**: replaced monolithic env.js with env.js (serverType
  one-liner, Makefile-managed) + keys.js (pre-computed crypto handles, generated
  by `384 init`) + config.js (unchanged profile selector). All imports across
  lib384/tests, cli/src, services/cli, and demos updated from env.js to keys.js.
  (AGENTS_DONE_TASKS/2026.03.31.001.FixEnvJsGeneration.md)

### Added

- `384 token` command: generates a correctly-formed storage token identifier
  (prefix LM2r + 32 random bytes, base62). Does not authorize — use `384 mint`
  for that.
- `384 init` now generates `~/.os384/keys.js` with full ChannelHandle objects
  (channelId and ownerPublicKey derived via SB384 async crypto).
- Standardized Makefiles (lib384, services) with `ENV=local|dev` enforcement.

### Removed

- `384 env` command (env.js is now a Makefile one-liner, keys.js generated by init).
- `lib384/env.example.js` (superseded by keys.js).

## [0.3.0-rc3] — 2025-03

- Initial RC3 release. Source migrated from lib-proto-03/cli.tools/. Cliffy-based CLI, Deno workspace import of lib384.
